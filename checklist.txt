Checklista zrealizowanych wymagań (zgodnie z tasks.txt):

1. Fundamenty koloru i skali
- [x] Gradient suwaka liczony w OKLCH, gamut-aware (sRGB).
- [x] Gradient zależny od L i C koloru bazowego oraz od roli koloru i trybu light/dark.
- [x] Symetryczna paleta względem koloru bazowego (anchor).
- [x] Różne krzywe jasności dla kroków jasnych i ciemnych.
- [x] Kolor bazowy zawsze w skali z unikalnym numerem (np. 414) i wyróżnieniem.

2. Suwaki i interakcja
- [x] Stabilne suwaki - brak resetów i utraty focusu dzięki renderowaniu różnicowemu.
- [x] Zmiana parametrów nie przebudowuje całego UI niepotrzebnie.

3. Kolory dodatkowe
- [x] Każdy kolor ma stały preview i rolę (dominant/secondary/accent).
- [x] Możliwość usuwania i zmiany kolejności (drag-and-drop).
- [x] Locki parametrów L i C.

4. Relacje kolorystyczne
- [x] System constraintów - limity dla relacji (triadic, complementary itp.).
- [x] Jasne komunikaty o przekroczeniu limitów i automatycznych korektach.

5. Palety specjalne
- [x] Paleta funkcjonalna (0/200/400/600/800) wizualnie odróżniona.
- [x] Paleta tęczowa (badge) o stałej liczbie hue.

6. UI / UX
- [x] Poprawna kolejność interakcji i layout (liczba kolorów NAD listą).
- [x] Walidacja HEX/RGB z wizualną informacją o błędzie.
- [x] Systemowe komunikaty o limitach i gamutach.

7. Role i hierarchia
- [x] Mapowanie ról (60-30-10) na logikę generowania chromy i jasności.

8. Accessibility
- [x] Sprawdzanie kontrastu AA / AAA na jasnym i ciemnym tle.
- [x] Dynamiczny kolor tekstu na swatchach (biały/czarny).
- [x] Brak wymuszonych auto-fixów.

9. Architektura i wydajność
- [x] Engine jako jedyne źródło prawdy (Single Source of Truth).
- [x] Brak mutacji stanu poza engine.
- [x] Memoizacja i wydajne renderowanie.

Wszystkie wymagania zostały zaimplementowane, przetestowane i zweryfikowane wizualnie.
