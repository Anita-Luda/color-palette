FINALNA CHECKLISTA ZGODNOŚCI Z WYMAGANIAMI

1. Palety Specjalne (Co 100 jednostek)
- [x] Paleta Funkcjonalna: kroki 0, 100, 200, 300, 400, 500, 600, 700, 800, 900, 1000.
- [x] Paleta Badge: kroki 0, 100, 200, 300, 400, 500, 600, 700, 800, 900, 1000.
- [x] Spójność: Wszystkie palety specjalne używają nowej logiki (0=Biały, 1000=Czarny).

2. Płynność Interakcji (Suwaki)
- [x] Real-time rendering: Suwaki działają na zdarzeniu `input`, nie `change`.
- [x] Płynne przesuwanie: Brak przeskakiwania (debounce usunięty dla UI), natychmiastowa reakcja palety.
- [x] Differential Rendering: Minimalizacja operacji na DOM podczas ruchu suwaka.

3. Wizualizacja Suwaków (Tracki)
- [x] Realistyczny podgląd: Gęste próbkowanie Hue (co 2 stopnie) na tracku.
- [x] Widoczność: Podniesiona wysokość tracka (18px), dodany kontrast i cienie.
- [x] Gamut-aware: Barwy poza sRGB są przyciemnione, dając jasny feedback użytkownikowi.

4. Dark Mode
- [x] Pełne przeliczenie: Przełączenie trybu aktualizuje tła i kontrast tekstu na swatchach.
- [x] Integracja: Systemowa flaga `palette_mode` steruje logiką silnika i stylem UI.

5. Hierarchia i Kolizje
- [x] Utrzymano system unikania kolizji (Collision Avoidance) dla Funkcyjnych i Badge.
- [x] Utrzymano nieliniowe skale Asymmetric.

System jest zoptymalizowany pod kątem wydajności i w pełni realizuje zlecony model inżynierski.
